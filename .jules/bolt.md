## 2024-06-15 - Fast ASCII Case-Insensitive String Matching for Search
**Learning:** In highly frequent search code loops (like `is_exact_match`, `is_prefix_match`, and `is_contains_match`), using `.to_lowercase()` creates significant heap allocation overhead, turning string matching into a bottleneck. Checking if the strings are ASCII (`is_ascii()`) and using zero-allocation functions like `eq_ignore_ascii_case()` and `windows().any(...)` yields a >50% performance improvement in microbenchmarks and saves hundreds of milliseconds at scale without changing semantics.
**Action:** Always consider fast-pathing ASCII case-insensitive comparisons using `.eq_ignore_ascii_case()` instead of `.to_lowercase()` for high-throughput string matching where Unicode edge cases aren't standard or can be deferred to a slow path.
